## 26B-A4B 서빙 명령어 (여러줄 설명)

bash에서 줄 끝에 `\`(백슬래시) 붙이면 다음 줄로 이어집니다.

```bash
source ~/gemma4venv/bin/activate && \
CUDA_VISIBLE_DEVICES=1 \
  python -m vllm.entrypoints.openai.api_server \
  --model /home/dongho.yoon/models/gemma-4-26B-A4B-it \
  --host 0.0.0.0 \
  --port 8001 \
  --served-model-name gemma-4-26B-A4B-it \
  --max-model-len 32768 \
  --enable-auto-tool-choice \
  --tool-call-parser gemma4 \
  --enforce-eager \
  --tensor-parallel-size 1 \
  --gpu-memory-utilization 0.7
```

각 옵션 설명:
- `source ~/gemma4venv/bin/activate` — Gemma4 전용 가상환경 활성화
- `CUDA_VISIBLE_DEVICES=1` — GPU 1번만 사용
- `--model ...` — 모델 경로
- `--host 0.0.0.0` — 외부(데스크탑)에서 접속 허용
- `--port 8001` — Qwen(8000)과 분리
- `--served-model-name` — API에서 쓸 모델 이름
- `--max-model-len 32768` — 최대 컨텍스트 길이 (32K)
- `--enable-auto-tool-choice` — tool call 자동 감지
- `--tool-call-parser gemma4` — Gemma4 전용 파서
- `--enforce-eager` — CUDA 그래프 비활성화 (호환성)
- `--tensor-parallel-size 1` — GPU 1장 사용
- `--gpu-memory-utilization 0.7` — VRAM 67GB 할당 (모델 49GB + KV캐시 18GB)

### temperature와 코딩 정확도
맞습니다. `temperature`가 모델의 "상상력"을 조절합니다.

- `temperature=0.0` — 항상 같은 답, 가장 결정적. **코딩에 추천**
- `temperature=0.2~0.4` — 약간의 변화, 적당한 창의성
- `temperature=0.7~1.0` — 다양한 답, 창의적 글쓰기용

이건 **서버 시작 옵션이 아니라 API 호출 시** 설정합니다:
```bash
curl http://localhost:8001/v1/chat/completions \
  -H 'Content-Type: application/json' \
  -d '{"model":"gemma-4-26B-A4B-it","messages":[{"role":"user","content":"hello"}],"temperature":0.0,"max_tokens":50}'
```

OpenCode/OmO에서는 `opencode.json`에서 설정 가능합니다.

`--dtype`(서버 옵션)은 VRAM 절약용이지 정확도와는 다른 개념입니다.

## vLLM Gemma4 듀얼 구성: 31B(GPU0) + E4B(GPU1)

Ollama 다운로드 기다리면서 vLLM으로 먼저 구성합니다.
Qwen 내리고 → 31B(GPU0) + E4B(GPU1) 동시 서빙.

---

### 1. Qwen vLLM 중지

```bash
# Qwen vLLM PID 확인
nvidia-smi | grep vLLM

# 또는 직접
ps aux | grep vllm | grep -v grep

# Qwen 프로세스 종료 (PID 확인 후)

```

### 2. GPU 비워졌는지 확인

```bash
nvidia-smi
```

### 3. gemma4venv 활성화 + 31B 서빙 (GPU 0, 포트 8000)

```bash
source ~/gemma4venv/bin/activate

# 31B 서빙 (GPU 0)

  --model ~/models/gemma-4-31b-it \

  --port 8000 \

  --max-model-len 32768 \
  --tool-call-parser gemma4 \
  --dtype bfloat16 \
  --trust-remote-code &

# 로딩 기다리기 (31B라 2~3분 걸릴 수 있음)
# 로그에서 "Uvicorn running" 뜨면 완료
```

### 4. E4B 서빙 (GPU 1, 포트 8001)

```bash
# E4B 서빙 (GPU 1)

  --model ~/models/gemma-4-E4B-it \
  --served-model-name gemma-4-E4B-it \

  --gpu-memory-utilization 0.4 \
  --max-model-len 32768 \

```

## 31B 재시작 — chat template + tool choice 수정

이전 서버는 시작 성공했고, 에러는 API 호출 시 발생한 것.
GPU 비워졌으니 바로 재시작합니다.

```bash
# GPU 비워졌는지 확인
nvidia-smi

# processor_config.json이 31B 폴더에 있는지 확인
ls ~/models/gemma-4-31b-it/processor_config.json

# 없으면 E4B에서 복사
cp ~/models/gemma-4-E4B-it/processor_config.json ~/models/gemma-4-31b-it/ 2>/dev/null

# 31B 서빙 (chat-template + enable-auto-tool-choice 추가)
source ~/gemma4venv/bin/activate

CUDA_VISIBLE_DEVICES=0 python -m vllm.entrypoints.openai.api_server \
  --model ~/models/gemma-4-31b-it \
  --served-model-name gemma-4-31b-it \
  --port 8000 \
  --gpu-memory-utilization 0.85 \
  --max-model-len 32768 \
  --enable-auto-tool-choice \
  --tool-call-parser gemma4 \
  --chat-template ~/gemma4venv/lib/python3.10/site-packages/vllm/transformers_utils/chat_templates/template_chatml.jinja \
  --dtype bfloat16 \
  --trust-remote-code &
```
